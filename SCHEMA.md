# SCHEMA — bamr87 dash

> Control plane for a fleet of ~40 project submodules: the root repo is the machinery (registry, gates, seeds, site) that manages and standardizes them.

## Conventions

- Structural facts live in the registry (`_data/projects.yml`); generated surfaces (README `AUTO:projects` span, `projects/SCHEMA.md` rows, build output) are regenerated, never hand-edited.
- `projects/*` are independent repos: the hub's pyramid stops at their boundary (`terminal`); each project carries its own pyramid, seeded from `templates/schema/`.
- Validate with `python3 tools/schema_lint.py check .` — wired into the drift gate.

## Placement

- New seedable artifact kit → `templates/<kit>/`, registered in `templates/SCHEMA.md`
- New gate, generator, or fleet script → `tools/`
- New operator doc → `docs/UPPERCASE-TOPIC.md`
- New registry → `_data/`, registered in `_data/SCHEMA.md`
- New project → `.gitmodules` + `_data/projects.yml`, then `tools/gen-projects-schema.py`

## Forbidden

- No hand edits to generated surfaces: `_site/`, `site/`, `Gemfile.lock`, the README AUTO span, or `projects/SCHEMA.md` rows.
- No project source code in the hub — it belongs in the project's own repo.
