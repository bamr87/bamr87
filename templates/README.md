# templates/

Scaffolding templates consumed by the dash's tooling — copied into target repos,
never built by Jekyll.

## `release-pipeline/`

The per-repo glue for the standardized release pipeline (see [`docs/RELEASES.md`](../docs/RELEASES.md)),
stamped into a repo by [`tools/adopt-release.sh`](../tools/adopt-release.sh):

| File | Purpose |
|------|---------|
| `ci.yml` | caller workflow → reusable `ci.yml@main` in `bamr87/.github` (`__DEFAULT_BRANCH__` is substituted) |
| `release.yml` | caller workflow → reusable `release-please.yml` + `publish.yml` |
| `RELEASING.md` | per-repo release cheat-sheet (`__REGISTRY__` / `__VERSION_SOURCE__` / `__SECRET__` substituted) |

The `release-please-config.json` + `.release-please-manifest.json` are generated by the
tool per detected ecosystem (`ruby` / `node` / `python` / `simple`) rather than templated here.

Placeholders use the `__NAME__` convention and are replaced by the tool via `sed`.
