# templates/

Scaffolding templates consumed by the dash's tooling — copied into target repos,
never built by Jekyll.

## `standard-ci/`

The baseline CI caller dropped into a submodule by
[`.github/workflows/standardize-fanout.yml`](../.github/workflows/standardize-fanout.yml):

| File | Purpose |
|------|---------|
| `ci.yml` | thin caller → the reusable `standard-ci.yml` gate in `bamr87/bamr87` (`__DEFAULT_BRANCH__` substituted) |

`standard-ci` is the minimal shared CI gate (lint + test + build, auto-detected);
`release-pipeline` below is the fuller release+CI adopted by publishable repos.

## `release-pipeline/`

The per-repo glue for the standardized release pipeline (see [`docs/RELEASES.md`](../docs/RELEASES.md)),
stamped into a repo by [`tools/adopt-release.sh`](../tools/adopt-release.sh):

| File | Purpose |
|------|---------|
| `ci.yml` | caller workflow → reusable `ci.yml@main` in `bamr87/.github` (`__DEFAULT_BRANCH__` is substituted) |
| `release.yml` | caller workflow → reusable `release-please.yml` + `publish.yml` |
| `RELEASING.md` | per-repo release cheat-sheet (`__REGISTRY__` / `__VERSION_SOURCE__` / `__SECRET__` substituted) |

The `release-please-config.json` + `.release-please-manifest.json` are generated by the
tool per detected ecosystem (`ruby` / `node` / `python` / `simple`) rather than templated here.

## `schema/`

The Pyramid Schema seed kit, stamped into a repo by
[`tools/seed-schema.sh`](../tools/seed-schema.sh) (locally) or
[`.github/workflows/schema-fanout.yml`](../.github/workflows/schema-fanout.yml) (PRs at fleet scale):

| File | Purpose |
|------|---------|
| `SCHEMA.template.md` | per-directory structural contract scaffold |
| `CLAUDE.snippet.md` | agent protocol section appended to the target's `CLAUDE.md`/`AGENTS.md` |
| `schema-check.yml` | CI gate running the vendored `schema_lint.py` (`__DEFAULT_BRANCH__` substituted) |
| `VERSION` | kit provenance (upstream commit, spec version, vendored date) |

## `agent-context/`

The agent-context kit, seeded by
[`.github/workflows/standardize-fanout.yml`](../.github/workflows/standardize-fanout.yml)
(artifacts `agent-context` and `claude`) via [`tools/fanout.sh`](../tools/fanout.sh) —
closes the `agent_context` requirement in [`_data/standards.yml`](../_data/standards.yml):

| File | Purpose |
|------|---------|
| `CLAUDE.template.md` | minimal CLAUDE.md scaffold (`__PROJECT_NAME__` / `__DEFAULT_BRANCH__` substituted); seeded only when the repo has **no** agent-context file |
| `claude.yml` | `@claude` mention workflow (`anthropics/claude-code-action@v1`, OAuth-first auth — see [`docs/AI-INTEGRATION.md`](../docs/AI-INTEGRATION.md)) |
| `VERSION` | kit provenance |

## `year-repo/`

Planned kit for the year-in-review knowledge-base repos (`1987`, `2005`, …) —
registered in `SCHEMA.md` but currently empty.

Placeholders use the `__NAME__` convention and are replaced by the tools via `sed`.
